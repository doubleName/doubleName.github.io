# vue-cli3打包后出现空白的解决方案

# 引用路径不对

第一种情况是打包后dist目录文件引用路径不对，会因为找不到问题而导致白屏。将默认路径是根目录(/)，改成当前目录(./)，再次npm run build就可以了。

在vue.config.js文件中加入如下配置：

module.exports = {
// 配置基本路径
  //vue-cli3.3以下版本使用
  // baseUrl: './'

  // vue-cli3.3+新版本使用
  publicPath: './' 
}

# 路由模式问题

由于路由模式默认是hash模式，如果设置为了history，打开就是不报错的空白。解决办法是改为默认hash模式。

const router = new Router({
    //mode改为hash，或是删除这条语句，默认也是hash模式
    mode: 'history',
    routes
});

|空白页面也反映了一些特殊需求，空白页面也不好看，在做项目的时候，最好载添加一个覆盖所有情况的候选资源：如果URL匹配不到任何静态资源，应该返回一个提示页面。
